# Excel SOP 版式焕新工具设计

## 1. 项目目标

开发一个可批量运行的 Excel SOP 版式焕新工具，将旧版洗衣机关键岗位 SOP 转换为新版标准格式。

第一阶段仅完成：

- 三份工作簿的结构分析。
- 字段、文字、图片和版式映射规则设计。
- “安装减震器螺栓”单岗位样例转换与验证。
- 精简冲突清单和处理日志设计。

单样例通过验收后，第二阶段再扩展为批量转换。

## 2. 输入与输出

### 2.1 输入文件

- `新版洗衣机SOP标准格式.xlsx`：新版标准模板和格式说明。
- `旧版滚筒关键岗位SOP.xlsx`：待焕新的旧版 SOP。
- `新版滚筒关键岗位SOP.xlsx`：已经完成的新旧版焕新对照案例。

### 2.2 第一阶段输出

- 单岗位新版 SOP 样例工作簿。
- 字段映射表。
- 图片定位与动态布局规则。
- 文字规范化规则。
- 精简冲突审核表。
- 转换处理日志。

所有结果另存为新文件，不覆盖源文件。

## 3. 处理原则

### 3.1 内容优先级

1. 旧版 SOP 中的工艺动作、质量要求、安全要求和关键参数。
2. 用户后续提供的明确修改指令。
3. 已完成新版案例中的布局和语言表达方式。
4. 新版空白模板中的固定格式和占位内容。

已完成新版案例只作为版式和措辞参考。案例与旧版发生实质冲突时，不使用案例覆盖旧版内容。

### 3.2 自动处理原则

- 能可靠推断的内容直接自动转换。
- 普通文字、标点、序号和版式问题自动处理并记录日志。
- 仅少量高风险冲突进入人工审核表。
- 用户可以在样例或批量转换后给出纠正指令，工具应支持定向重跑。

## 4. 字段映射

| 旧版字段 | 新版字段/区域 | 映射规则 |
|---|---|---|
| 文件编号 | 文件编号 | 原值保留 |
| 适用型号 | 产品名称 | 原值保留，仅规范无意义空格 |
| 工序名称 | 岗位名称 | 旧版原值优先 |
| 工序号 | 岗位编号 | 按可维护的岗位编码规则转换；无法确认时保留旧值并标记 |
| 生产节拍 | 作业时间 | 数值和单位原样保留 |
| 工序定员 | 作业人数 | 直接映射 |
| 生效日期、更改履历 | 制定日期、修订日期、版次 | 最早有效日期作为制定日期，最近变更日期作为修订日期；版次按明确记录推导 |
| 工装名称、用量 | 设备/工装/工具要求 | 分别填写名称、数量和设置说明 |
| 物料名称、用量 | 物料明细 | 保留名称、规格、型号和数量 |
| 操作图示说明 | 作业步骤 | 按动作及图片关系重新分组和编号 |
| 本岗位注意事项 | 控制点、质量要求 | 操作方法归入控制点；参数和验收标准归入质量要求 |
| 雷区、触雷后果 | 相关步骤的控制点或质量要求 | 归入对应步骤，避免重复堆叠 |
| 更改履历 | SOP 变更记录 | 保留日期、内容和原因 |

字段值存在冲突时，使用内容优先级规则，不静默覆盖。

## 5. 中间数据模型

每个岗位先转换为与 Excel 版式无关的结构化数据：

```text
岗位
├─ 基本信息
│  ├─ 产品名称
│  ├─ 适用线体
│  ├─ 岗位名称
│  ├─ 岗位编号
│  ├─ 作业人数
│  ├─ 作业时间
│  ├─ 文件编号
│  └─ 日期与版次
├─ 工装列表
├─ 物料列表
├─ 作业步骤
│  ├─ 步骤编号
│  ├─ 动作描述
│  ├─ 图片列表
│  ├─ 控制点
│  ├─ 质量要求
│  └─ 风险标记
├─ 变更记录
└─ 冲突项
```

文字识别、图片匹配和动态排版均围绕该模型进行，避免把旧版坐标直接硬编码为新版坐标。

## 6. 作业步骤识别

### 6.1 动作分组依据

综合使用以下信息识别完整动作：

- 文本框的空间区域和顺序。
- 图片在旧版工作表中的锚点位置。
- 箭头、①②③等顺序标记。
- 动词和操作对象的连续性。
- 图片内容是否属于同一操作对象。
- 同一区域内文字与图片的邻近关系。

同一段文字中的“拿取—检查—安装—自检”可以拆为多个步骤，也可以保留为一个完整动作。判断标准是：拆分后是否改变原工艺动作含义，以及图片是否存在明确的一一对应关系。

### 6.2 连续编号

- 模板中的预设编号只作为占位符。
- 转换后按照实际作业顺序从 1 开始连续编号。
- 同一动作的多张图片不会因模板槽位限制而拆成多个步骤。
- 跨页时编号继续递增，不重新从 1 开始。

## 7. 图片识别与定位

### 7.1 图片提取

- 从 `.xlsx` 包中的图片对象和绘图锚点直接提取。
- 不依赖屏幕截图或人工复制。
- 计算图片哈希，用于识别重复图片、模板图标和签名图片。
- 保留图片原始像素尺寸和宽高比。

### 7.2 图片归属

图片根据以下顺序归入步骤：

1. 明确的①②③标记或箭头关系。
2. 与文本框共享或相邻的旧版版面区域。
3. 从上到下、从左到右的阅读顺序。
4. 操作对象和图片内容的一致性。

只有无法可靠归属、可能重复或存在多个合理归属时，才进入高风险冲突清单。

### 7.3 图片排版

- 始终等比例缩放，不拉伸。
- 默认完整显示，不裁切关键内容。
- 多张图片按照实际作业顺序排列。
- 根据图片比例自动选择横排、竖排或网格。
- 不为填满模板而过度放大低清晰度图片。
- 不为适应单页而过度压缩图片；空间不足时增加页面。

## 8. 动态布局

### 8.1 固定区域

以下区域继承新版模板，不允许被作业步骤占用：

- 标题和岗位基本信息。
- 设备/工装/工具要求。
- 物料明细。
- 控制点和质量要求。
- SOP 变更记录。
- 操作标识和签核区域。

### 8.2 作业步骤槽位

作业步骤区域使用可合并的基础槽位，不机械使用固定的 1、2、3、4 四个区域。

建议的初始容量模型：

- 单图短文本：一个基础槽位。
- 双图或较长文本：横向合并两个基础槽位。
- 三至四张图片：双槽位或整行。
- 超长说明：优先扩大文字区，不压缩图片到不可辨识。

最终槽位尺寸由整份 SOP 的步骤数、图片数、图片比例和文字量共同决定。

### 8.3 整体规划

排版前必须：

1. 统计全部步骤。
2. 统计每一步的图片数量、比例和最低清晰尺寸。
3. 估算文字所需高度。
4. 计算一页可安全容纳的组合。
5. 再统一分配槽位并生成页面。

禁止边读取边排版，因为这会造成前面步骤占用过多空间、后续步骤被压缩或遗漏。

### 8.4 多页处理

- 单页无法容纳时，复制新版模板的页面结构。
- 新页面继续使用相同字体、边框、列宽、行高和页面参数。
- 每页保留必要的固定区域。
- 步骤编号跨页连续。
- 页面之间不得重复或遗漏步骤。

## 9. 文字规范化

### 9.1 可以自动修改

- 统一标点、空格、序号和换行。
- 统一参数单位的书写格式，但不改变数值。
- 将描述整理为“操作对象＋操作动作＋自检标准”。
- 删除完全重复的句子。
- 调整语序，使动作顺序更清晰。
- 将操作要求归入控制点，将关键参数和验收标准归入质量要求。

### 9.2 禁止自动修改

- 型号、数量、扭矩、压力、电压、间隙和时间等参数。
- 工艺动作顺序。
- 物料和工具的实质名称。
- 适用产品或适用线体。
- 原文没有表达的新增工艺要求。

“螺栓/螺钉”“电枪/气动枪”“减震器/减振器”等可能影响工艺含义的词，不因语言统一而擅自替换。

## 10. 冲突处理

### 10.1 自动处理

以下问题自动处理，不进入人工审核表：

- 标点、空格和编号不统一。
- 不影响含义的语序调整。
- 模板占位编号更新。
- 图片等比例缩放和版面位置调整。
- 可由明确规则判断的字段搬运。

### 10.2 高风险冲突

仅以下情况进入人工审核表：

- 关键参数缺失或在不同来源间矛盾。
- 岗位名称、编号、物料或工具发生实质冲突。
- 图片无法可靠归入某一步骤。
- 有文字无对应图片或有图片无对应动作，且可能导致步骤遗漏。
- 现有页面无法完整容纳且自动分页仍存在版面风险。

### 10.3 冲突审核表

审核表至少包含：

| 字段 | 用途 |
|---|---|
| 岗位 | 定位 SOP |
| 步骤 | 定位具体步骤 |
| 冲突类型 | 参数、工具、图片、字段或步骤 |
| 旧版值 | 默认可信来源 |
| 案例值 | 对照参考 |
| 自动采用值 | 当前转换结果 |
| 风险说明 | 为什么需要关注 |
| 最终采用值 | 用户可填写 |
| 处理指令 | 保留、替换、忽略或其他明确指令 |

审核表只收录高风险事项，避免把最终审核变成逐字段确认。

## 11. 样例岗位处理设计

第一阶段样例为“安装减震器螺栓”。

已识别的旧版主要内容包括：

- 岗位名称：紧固减震器螺栓。
- 工序号：GT-HZ03。
- 作业人数：1。
- 生产节拍：15s。
- 物料：减震器螺栓 M10*43.5，数量 2。
- 工装：ETV DS72-30-10 电枪，数量 1。
- 关键力矩：20–25 N·m。
- 操作包括拿取、检查、安装和紧固减震器螺钉。
- 安装减振器时避免直接握拿钢管，防止蹭掉润滑油。

已完成新版案例存在以下冲突：

- 岗位名称填写为“吊筒”。
- 工具改为“固定扭矩气动枪”。
- 关键力矩 20–25 N·m 未完整保留。

样例转换时：

- 使用旧版岗位名称、工具和关键参数。
- 使用新版案例的动态合并布局作为参考。
- 将案例冲突写入精简冲突表，但不阻止样例生成。

## 12. 技术架构

### 12.1 模块划分

1. 工作簿解析器  
   读取单元格、合并区域、图片、文本框、样式和页面设置。

2. 旧版内容提取器  
   将旧版字段、动作、图片、物料、工具和要求转换为中间数据。

3. 规则与冲突引擎  
   执行字段映射、动作分组、文字规范化和风险判断。

4. 布局规划器  
   估算步骤空间，选择槽位组合并规划分页。

5. 新版工作簿生成器  
   复制模板结构、填写内容、插入图片并保持模板格式。

6. 验证器  
   检查步骤、图片、关键参数、固定区域和打印版面。

7. 审核与日志输出器  
   生成精简冲突表和处理日志。

### 12.2 数据流

```text
旧版工作簿
  → 结构解析
  → 中间数据
  → 规则映射与冲突检测
  → 整体布局规划
  → 新版模板生成
  → 内容与视觉验证
  → 样例工作簿＋冲突表＋日志
```

## 13. 验收标准

### 13.1 内容完整性

- 旧版有效步骤无遗漏。
- 旧版有效图片无遗漏、无重复。
- 图片与步骤无明显错配。
- 物料、工具和数量完整。
- 工艺、安全、质量要求完整。
- 未经用户指令，关键参数不得改变。

### 13.2 版面质量

- 图片无拉伸和明显失真。
- 图片内容清晰可辨。
- 同一动作的图片和文字处于同一区域。
- 步骤编号连续且顺序正确。
- 固定区域未被破坏或覆盖。
- 字体、边框、列宽、行高和模板风格保持一致。
- A3 横向打印时无关键内容截断。
- 单页容量不足时正确增加页面。

### 13.3 可维护性

- 映射规则和冲突规则可配置。
- 用户纠正后可定向重新处理单岗位。
- 批量转换不得依赖手工逐页排版。
- 每次转换均能追溯自动处理和用户指令。

## 14. 第一阶段完成边界

第一阶段在以下条件全部满足后完成：

- 设计和规则经用户确认。
- “安装减震器螺栓”样例生成。
- 样例通过内容完整性、图片、版面和打印检查。
- 冲突表数量精简且可操作。
- 用户确认样例方向可扩展到其他岗位。

此后才进入批量转换阶段。
